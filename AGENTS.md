# AGENTS.md — 剧本创作辅助工具

## 项目阶段

**当前处于设计阶段，没有任何实现代码。** 仓库内容为 OpenSpec 变更文档和技能定义，不可修改 CLAUDE.md 中的核心原则。

## 核心原则（来自 CLAUDE.md，不可覆盖）

CLAUDE.md 是项目的哲学基石。所有交互必须遵守：

- **创作永远没有绝对的正确。** 工具是辅助者，不是裁决者。
- **禁止**用理论否定用户创意、评判想法好坏、证明用户"想错了"
- **应该**理解用户意图，用知识滋养创意，帮用户走通自己的路
- 理论是工具不是答案，案例是参考不是模板

不要在 AGENTS.md 中重复 CLAUDE.md 内容——直接引用即可。

## 产品定位

本工具定位为**剧作老师**——不是项目管理者（OhMyOpenAgent 模式），也不是任务执行器（ClaudeCode 模式）。

**核心循环**：用户自由创作 → 后台观察/提取数据 → 判断进度和矛盾 → 温和提醒 → 用户决定 → 继续循环

- 不强制流程、不评判想法、不打断创作
- 五个模块（世界/角色/情节/主题/结构）螺旋迭代，不是线性完成
- `画板文档架构.md` 记录了最新的架构草图，**其中关于前端拆分、Agent 调度、四数据库的规划比 `design.md` 中的技术栈描述更接近当前设计意图**

## 架构要点（⚠️ 实现时必须遵守）

### Agent 系统：两层结构

| 层 | 定位 | 触发方式 | 做什么 |
|---|---|---|---|
| **第一层：响应式 Agent** | 用户说话就响应 | 意图分类 → 路由 | 数据提取、知识检索、内容生成、格式转换、矛盾检测等 |
| **第二层：主动式状态机** | 后台持续运行 | 数据变化 → 判断 → 提醒 | 进度追踪、引导决策、矛盾监控、伏笔追踪、质量感知 |

**关键约束**：两层之间需要上下文感知——状态机在提醒之前必须知道"用户现在在干什么"、"是不是刚提醒过了"、"现在打断合适吗"。避免状态机的主动提醒和响应用户的 Agent 互相打架。

### 数据库：四种并存（不是单一 SQLite）

- **向量数据库**：RAG 知识库检索
- **JSON 数据库**：五大模块数据（世界/角色/情节/主题/结构）+ 知识库内容
- **传统数据库**：项目元数据、结构化配置
- **内存数据库**：热数据缓存

### 前端 UI：六个独立模块

监控台、文本编辑器、Chat UI、画布 UI、存档树 UI、设置 UI（详见 `画板文档架构.md`）

## OpenSpec 工作流

项目使用 OpenSpec（v1.3.1）管理变更，schema 为 `spec-driven`。四条核心命令：

| 命令 | 用途 |
|------|------|
| `/opsx:explore` | 探索想法、理清需求——只思考不实现 |
| `/opsx:propose` | 创建变更提案（proposal + design + tasks） |
| `/opsx:apply` | 按 tasks.md 逐项实现 |
| `/opsx:archive` | 完成后归档到 `openspec/changes/archive/` |

当前唯一活跃变更：`script-writing-assistant`（剧本创作辅助工具）
