# 剧本创作辅助工具 — 设计文档 v0.1

> 2026-06-04 设计探索阶段成果汇总
> 尚未进入实现阶段，以下内容为架构方向共识

---

## 一、核心哲学

来自 CLAUDE.md，不可修改：

- 创作永远没有绝对的正确
- 工具是辅助者，不是裁决者
- 用知识滋养创意，帮用户走通自己的路
- 理论是工具不是答案，案例是参考不是模板

---

## 二、产品定位

**剧作老师**——不是项目管理者，不是任务执行器。

核心循环：用户自由创作 → 后台观察/提取数据 → 判断进度和矛盾 → 温和提醒 → 用户决定 → 继续循环

- 不强制流程、不评判想法、不打断创作
- 五个模块（世界/角色/情节/主题/结构）螺旋迭代

---

## 三、交互模型

### 主入口：ChatUI

用户主要通过 ChatUI 与工具聊天创作。支持：
- 渐进式提问（由浅入深，不一次抛出一堆问题）
- 叠甲提醒（五层免责保护下的专业建议）
- 心流模式（几乎不打扰用户）

### 画布UI（只读查看）

用户可切换到画布查看五大模块的结构化数据：
- 世界观模块 / 角色模块 / 情节模块 / 主题模块 / 结构模块

### 卡片系统

用户可选添加监控卡片到侧边栏，实时显示 Agent 摘要：
- 矛盾监控卡片、节奏分析迷你图、提取状态指示等
- 非侵入式，不打断创作

### 排队消息

在当前对话间歇，按优先级排队提醒用户：
- P1: 用户主动触发的生成结果
- P2: 逻辑硬伤
- P3: 顽固模式（持续未解释的矛盾）
- P4: 重大涟漪（核心设定变更波及多个模块）
- P5: 普通提醒

### 独立报告

低频大块头输出，在保存时/下次打开时展示：
- 更改汇总（上次离开后的变化）
- 节奏全局评估
- 灵感推荐整理

## 四、Agent 架构

### 导播室 Hub（Agent Hub）

所有 Agent 通过 Hub 中转，互不直接通信。如同片场的导演——
每个部门只管干活，导演决定哪个画面优先、什么时候切。

Agent 只做两件事：
1. 向 Hub 提交"事件"（发现了什么）
2. 从 Hub 接收"指令"（现在该做什么）

### 第一层：响应式 Agent（用户说话即响应）

| Agent | 角色 | 职责 |
|-------|------|------|
| 对话 Agent | 摄影指导 | 聊天、渐进引导、叠甲提醒、输出排队消息 |
| 提取 Agent | 灯光师 | 聊天→结构化数据，轻量实时 + 深度空闲 |
| 生成 Agent | 编剧组 | 大纲扩写、角色小传、场景描述等 |
| 格式 Agent | DIT/输出 | Fountain 格式转换，输出硬门禁（否决权） |

### 第二层：主动式状态机（后台常驻）

| Agent | 角色 | 职责 |
|-------|------|------|
| 矛盾监控 Agent | 场记 | 四维度连续性检查：角色/时间线/世界观/情节 |
| 涟漪检测 Agent | 统筹 | 新内容影响扩散分析，触发相关模块更新 |
| 节奏分析 Agent | 剪辑指导 | 情绪曲线、结构评估，阶段报告 |
| 风格守护 Agent | 美术指导 | 跨章节语气一致性、角色对话风格追踪 |
| 伏笔追踪 Agent | 剧本统筹 | 播种→提醒→回收全生命周期管理 |
| 灵感推荐 Agent | 资料库 | 从知识库找滋养创意的参考（案例/理论/旧项目） |
| 时机管理 Agent | 副导演 | 层甲决策、心流检测、响应频率控制 |
| 记忆管理 Agent | 场记组长 | 用户偏好更新、空闲总结、更改汇总 |

### Agent 间通信原则

- Agent 之间不直接通信，所有事件提交到 Hub
- Hub 统一决策：什么时候说、说的顺序、说到哪里、说的厚度
- 类比：片场各部门不会互相喊话，所有条子经过导演的手

## 五、知识库

### 技术底座

RAG-Anything（HKUDS 实验室开源，MIT 协议）：
- 多模态文档摄入 → 知识图谱构建 → 混合检索（向量 + 图遍历）
- 支持三层知识图谱：实体层 + 事件因果层 + 主题抽象层
- 这从根本上区别于其他剧本工具——不只是"找到相似段落"，而是理解因果和关系

### 知识库内容（待详细设计）

- 故事库：用户自己的旧项目存档
- 剧作理论库：麦基、坎贝尔、菲尔德等
- 规则库：专业知识体系
- 案例库：公共领域剧本、大纲等
- 视觉参考库：预留接口，前期不实现

## 六、用户记忆文档

区分于知识库——这是"关于用户"的数据：

**用户级（跨项目）**：创作节奏、时段偏好、术语熟悉度、互动习惯

**项目级（单项目）**：各分析维度权重、已免疫的提醒类型、薄弱环节标记

作用：驱动 Agent 的自适应行为——什么时候提醒、用什么语言、叠几层甲。

---

## 七、螺旋迭代与版本

- 用户自己决定保留什么（可能只留剧本名）
- 版本是生长，不是删改
- 可从任意历史版本"分叉"开新分支
- 跨分支比较和部分合并

## 八、核心差异化机制

### 叠甲提醒（五层）
1. 免责声明（"按传统视角…"）
2. 具体事实，不下结论
3. 可能原因，不是唯一答案
4. 后果展示，非评判
5. 退路保留（"如果是先锋艺术…"）

### 四维度连续性检查
- 角色连续性（默认权重 90%）
- 时间线连续性（默认权重 85%）
- 世界观一致性（默认权重 75%，先锋艺术降权）
- 情节连续性（默认权重 70%，先锋艺术降权）

### 心流模式
最高优先级的静默状态，压倒一切——再严重的问题也只亮卡片，不插队。

### 顽固模式检测
单次异常 → 一问；持续出现且无解释 → 升级提醒；严重逻辑矛盾 → 红灯 + 排队消息。

### 格式否决权
纯技术格式问题（缺少角色名、边距错误等），非创作评判。格式 Agent 在导出时硬拦截。
