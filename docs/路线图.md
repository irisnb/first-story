## 当前节点说明

### 当前在做什么？

当前不是在做 UI，也不是在做 Agent，更不是在做知识库。

当前在做的是：

> **验证地基图纸能不能搭起来。**

也就是补：

```text
openspec/changes/define-minimal-story-state-and-event-log/walkthrough.md
```

### 为什么 UI 还没开始？

因为 UI 要展示的数据还没被真实例子验证。

如果现在就画完整 UI，会遇到这些问题：

- ChatUI 要显示什么状态？取决于 `story_state`。
- 提醒卡片引用什么证据？取决于 `ContinuityEvent → Fact`。
- 用户点“忽略”后改什么？取决于 `UserPreference`。
- 画布展示哪些字段？取决于 `Character / PlotEvent / Fact` 是否稳定。

所以顺序不是“先画完整 UI”，而是：

```text
walkthrough.md 跑通
  ↓
字段和证据链稳定
  ↓
画第一张“最小 UI 闭环图”
```

## 当前不要做什么

为了防止项目再次横向铺开，当前先不要做：

- 不开第二个 OpenSpec change。
- 不画完整六大 UI 模块。
- 不接真实 LLM。
- 不接 LightRAG。
- 不选完整技术栈。
- 不做版本树、导出、心流自动识别。

当前只问一个问题：

> **“姐姐十年前死亡却昨天打电话”这块积木，能不能按图纸搭起来？**

## 更新规则

这张路线图不是圣旨，可以改。

更新规则：

1. 每次完成一个节点，更新“当前节点”。
2. 如果 walkthrough 或实现证明路线不对，允许改节点顺序。
3. 不在这里记录所有细节；细节进对应 OpenSpec change。
4. 不在这里堆问题；问题进 `docs/问题清单-待解决.md`。
5. Later 扩展只有在 MVP 稳定后才拆新 change。
