## 当前节点说明

### 当前刚完成的是什么？

当前刚完成的是：

> **验证并修正地基图纸能不能搭起来。**

也就是完成并修正：

```text
openspec/changes/define-minimal-story-state-and-event-log/walkthrough.md
```

修正后的重点包括：忽略不等于确认设定、项目偏好必须经 event log 入账、`event_id` 与 `idempotency_key` 拆分、Fact 补原文锚点。

### 为什么 UI 还没开始？

因为 UI 要展示的数据已经被真实例子验证，但还没进入最小后端骨架的实现阶段。

如果现在就画完整 UI，会遇到这些问题：

- ChatUI 要显示什么状态？取决于 `story_state`。
- 提醒卡片引用什么证据？取决于 `ContinuityEvent → Fact`。
- 用户点“忽略”后改什么？取决于 `project_preferences` 的边界。
- 画布展示哪些字段？取决于 `Character / PlotEvent / Fact` 是否稳定。

所以顺序不是“先画完整 UI”，而是：

```text
walkthrough.md 跑通并修正
  ↓
字段和证据链稳定
  ↓
开始最小后端骨架
```

---

## 当前不要做什么

为了防止项目再次横向铺开，进入最小后端骨架前先不要做：

- 不画完整六大 UI 模块。
- 不接真实 LLM。
- 不接 LightRAG。
- 不选完整技术栈。
- 不做版本树、导出、心流自动识别。

当前只问一个问题：

> **能不能把已经验证过的 event log + story_state 施工图做成最小可跑后端？**

## 更新规则

这张路线图不是圣旨，可以改。

更新规则：

1. 每次完成一个节点，更新“当前节点”。
2. 如果 walkthrough 或实现证明路线不对，允许改节点顺序。
3. 不在这里记录所有细节；细节进对应 OpenSpec change。
4. 不在这里堆问题；问题进 `docs/问题清单-待解决.md`。
5. Later 扩展只有在 MVP 稳定后才拆新 change。
