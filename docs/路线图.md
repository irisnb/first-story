## 当前节点说明

### 当前刚完成的是什么？

当前刚完成的是：

> **最小后端骨架 + P0 安全护栏。**

具体包括：

- event log 追加和重放
- story_state 投影重建
- CORS 收窄（localhost only）
- payload 校验（事件类型 + 字段）
- replay fail-fast（坏日志抛异常而非静默跳过）
- 代码卫生（ruff clean）

### 下一步做什么？

> **做最小写作闭环。**

这是一个端到端的产品闭环，不是单纯的后端或 UI：

```text
[用户写故事] 文本编辑器（极简）
      ↓
[保存正文] document revision API
      ↓
[提取事实] 提取 Agent → Fact
      ↓
[生成证据卡] 矛盾监控 → ContinuityEvent
      ↓
[用户处理] 忽略/接受/保留 → project_preferences
```

### 为什么这是下一步？

五路 reviewer 的核心判断：

> 现在最大问题不是后端有没有，而是产品闭环和信任机制还没成型。

后端骨架是"地基"，但用户不能在地基上写故事。下一步必须让用户能：

1. **写**：有一个文本编辑器（不是后台操作面板）
2. **看**：系统能提取事实、发现矛盾、生成提醒
3. **选**：用户能忽略/接受/保留，系统记住偏好

---

## 当前不要做什么

为了防止项目再次横向铺开，做最小写作闭环时不要：

- 不画完整六大 UI 模块（先做一个极简文本编辑器 + 侧边证据卡）
- 不接 LightRAG（灵感检索是 Later 扩展）
- 不接 LangGraph 多 Agent（当前只需提取 + 矛盾监控两个核心能力）
- 不做团队协作（首发人群是单人编剧）
- 不做完整导出/版本树（先让闭环跑通）

当前只问一个问题：

> **用户能不能完成一次"写作 → 提醒 → 选择"的完整闭环？**

## 更新规则

这张路线图不是圣旨，可以改。

更新规则：

1. 每次完成一个节点，更新"当前节点"。
2. 如果 walkthrough 或实现证明路线不对，允许改节点顺序。
3. 不在这里记录所有细节；细节进对应 OpenSpec change。
4. 不在这里堆问题；问题进 `docs/问题清单-待解决.md`。
5. Later 扩展只有在 MVP 稳定后才拆新 change。
