## Context

当前系统架构：
- 用户通过 ChatUI 发送消息
- 对话 Agent 调用 LLM 生成回复
- 提取 Agent 将内容提取到扁平列表（facts[], plot_events[], characters[]）
- 投影器从事件日志重建 story_state.json

问题：
1. 无分类：所有"事实"混在一起
2. 无文档结构：只是条目列表
3. 用户无法直接编辑
4. 不符合用户预期的"不断完善的世界观文档"形式

设计文档参考：`docs/五兄弟重构设计文档.md`

## Goals / Non-Goals

**Goals:**
- 五模块 MD 文档作为用户可见的真相源
- 独立的分类 API，判断内容归属
- MD 文档解析器，同步到事件日志
- 乐观锁机制，用户编辑优先
- 前端画布 UI 展示五模块

**Non-Goals:**
- 用户自定义 section（V1 使用固定 section）
- 跨项目模块模板
- 实时协作编辑（单用户场景）

## Decisions

### 决策 1：真相源 = MD 文档

**选择**：MD 文档是用户可见的真相源，事件日志是派生数据

**数据流**：
```
MD 文档（用户编辑）→ 解析器 → 事件日志 → 投影 → story_state.json
```

**理由**：
- 用户看到/编辑的是 MD（真相源）
- Agent 工作用的是事件日志（结构化）
- 两者通过解析器同步

**替代方案**：事件日志为真相源 → 用户编辑后同步到 MD
- 否决理由：用户心智模型是"我在编辑文档"

### 决策 2：分类 API 独立

**选择**：独立的分类 API，每条 candidate 消息触发

**理由**：
- 单一职责，prompt 更清晰
- 失败隔离
- 可用不同模型（分类用便宜模型）

**成本**：每条消息增加 ~550 tokens（约 70% 增加）

### 决策 3：固定 section + LLM 判断

**选择**：每个模块有预定义的 section 列表，LLM 判断内容放哪个 section

**模块 Section 定义**：
- **world.md**: 总述、魔法/技术系统、社会结构、地理、时间线锚点、细节记录
- **characters.md**: 总述、主要角色、次要角色、角色关系
- **plot.md**: 总述、主线、支线、关键事件
- **theme.md**: 总述、核心主题、次要主题
- **structure.md**: 总述、幕布结构、节奏设计、关键转折点

### 决策 4：乐观锁

**选择**：用户编辑时锁定文档（revision + checksum），系统添加排队等待

**锁机制**：
- TTL = 5 分钟
- 每 30 秒心跳续期
- 锁过期后可被抢占

### 决策 5：追加格式 = 列表项

**选择**：系统追加内容时，以 Markdown 列表项形式追加到对应 section

**格式**：`- 内容描述`

### 决策 6：MD 解析器 = Python markdown-it

**选择**：使用 Python markdown-it-py 库解析 MD 文档

**理由**：
- Python 后端原生支持
- 结构化程度高
- 已有成熟生态

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| MD 解析器复杂度高 | 使用成熟的 markdown-it-py 库，定义清晰的模板格式 |
| 用户编辑破坏结构 | UI 层校验结构完整性，提供格式提示；后端容错解析 |
| 分类准确率不够 | 调优 prompt，收集 bad case 迭代 |
| 成本增加 70% | 分类可用更便宜的模型，只对 candidate 消息调用 |
| 编辑冲突 | 乐观锁 + 队列，用户优先 |

## Migration Plan

### Phase 1: 基础设施
1. 定义五模块 section 结构
2. 实现 MD 解析器（markdown-it-py）
3. 实现 MD 渲染器（追加列表项）

### Phase 2: 分类 API
1. 设计 prompt
2. 实现端点
3. 测试分类准确率

### Phase 3: 集成
1. 修改对话流程，每条 candidate 消息触发分类
2. 分类结果写入 MD 文档
3. MD 变更触发解析 → 事件日志

### Phase 4: 乐观锁
1. 锁 API
2. 前端编辑器集成
3. 队列处理

### Phase 5: 前端 UI
1. 五模块文档编辑器
2. ModulesView 改造
3. 文档预览

### 回滚策略
- 保留旧的事件日志格式
- 新旧数据模型并行期间，可快速切回

## Open Questions

1. 分类 API 是否需要返回"不分类"选项？（有些内容可能不属于任何模块）
2. 用户删除 MD 中的系统添加内容时，是否需要同步删除事件日志？
3. 数据迁移：现有扁平列表数据如何迁移到新格式？
