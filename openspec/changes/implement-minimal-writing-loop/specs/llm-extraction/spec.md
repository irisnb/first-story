## ADDED Requirements

### Requirement: Fountain 结构解析确定角色
提取能力 SHALL 通过 Fountain 结构解析（角色名为对白前的居中大写行）确定剧本角色集合，不通过启发式猜测或 NER 推断角色。

#### Scenario: 从对白结构识别角色
- **WHEN** 正文包含若干角色名行及其后续对白
- **THEN** 系统将这些角色名识别为角色，并将对白归属到对应角色

#### Scenario: 不把动作行词语当角色
- **WHEN** 动作行中出现像人名的普通名词（如"房间""夜晚"）
- **THEN** 系统不将其识别为角色，因为它不处于 Fountain 角色名位置

### Requirement: LLM 语义提取事件与状态
提取能力 SHALL 将确定性结构（场景、角色、对白归属）作为上下文喂给 LLM，由 LLM 提取事件摘要、角色状态变化与事实断言，输出为结构化 Fact。

#### Scenario: 提取事件与状态变化
- **WHEN** 提取能力处理一段包含情节进展的正文
- **THEN** 系统调用 LLM 产出结构化 Fact（含事件摘要、涉及角色、状态变化），而非原始自然语言

#### Scenario: 确定性结构不消耗 LLM 调用
- **WHEN** 提取场景标题、角色集合、对白归属等可由 Fountain 解析确定的信息
- **THEN** 系统直接由解析器产出，不为这部分发起 LLM 调用

### Requirement: 提取攒批触发且失败不阻断
提取能力 SHALL 在停笔/保存后批量触发，且 LLM 调用失败或超时时 MUST NOT 阻断用户写作。

#### Scenario: 保存后批量提取
- **WHEN** 用户保存正文
- **THEN** 系统将提取任务排入后台批处理，不在用户每次键入时逐字提取

#### Scenario: 提取失败不阻断创作
- **WHEN** LLM 提取调用失败或超时
- **THEN** 系统记录失败但允许用户继续写作，并在下次触发时补做提取
