## ADDED Requirements

### Requirement: 三抽屉导航结构
前端 SHALL 提供左侧竖排导航的三抽屉结构：① 主界面（ChatUI 为主角 + 证据栏常驻 + 直达正文按钮）、② 项目设置二级页（项目选择/版本/信息/风格备忘）、③ 五兄弟二级页（世界观/角色/剧情/主题/结构入口，可进三级详情页）。另有独立的正文二级页。

#### Scenario: 打开应用进入主界面
- **WHEN** 用户打开应用并选中一个项目
- **THEN** 默认进入主界面，ChatUI 作为主角占据主区域
- **AND** 证据栏常驻可见，并提供直达正文页的按钮

#### Scenario: 切换抽屉
- **WHEN** 用户点击左侧竖排导航的某个抽屉
- **THEN** 界面切换到对应的二级页（项目设置 / 五兄弟 / 正文）
- **AND** 主界面 ChatUI 不被销毁，可随时切回

#### Scenario: 进入五兄弟详情
- **WHEN** 用户在五兄弟二级页点击某个模块入口
- **THEN** 进入该模块的三级详情页（世界观/角色/剧情/主题/结构之一）

### Requirement: ChatUI 是主角且气泡非模态
ChatUI SHALL 是产品主入口，占据主界面主区域。聊天气泡 SHALL 只显示对话内容，MUST NOT 拦截或改写用户输入。矛盾/连续性证据 MUST NOT 出现在对话气泡里。

#### Scenario: 发送聊天消息
- **WHEN** 用户在 ChatUI 输入并发送消息
- **THEN** 用户原文与对话 Agent 回复以气泡形式呈现
- **AND** 用户输入不被拦截或改写

#### Scenario: 意图态提示
- **WHEN** `/chat` 响应返回 `intent`
- **THEN** 前端可据此给出轻量提示（如 candidate 提示"已记为想法"）
- **AND** 该提示非模态、非告警红色

### Requirement: 证据只走证据栏、非告警呈现
矛盾/连续性证据 SHALL 仅通过证据栏（轮询 story_state）渲染，MUST NOT 出现在对话气泡里。证据栏 SHALL 非模态、非告警红色，符合"只递证据不报警"原则，排版用心。

#### Scenario: 聊天触发矛盾检测
- **WHEN** 用户聊天内容触发了矛盾检测并产生证据
- **THEN** 证据出现在证据栏，而非对话气泡
- **AND** 证据栏展示非模态、非告警红色

### Requirement: 放入正文按钮
ChatUI SHALL 提供"放入正文"按钮，触发采纳进正文 endpoint，把选定内容追加到正文末尾。此按钮与后端采纳 endpoint 成对存在。

#### Scenario: 点击放入正文
- **WHEN** 用户在 ChatUI 选定内容并点"放入正文"
- **THEN** 前端调用采纳 endpoint
- **AND** 采纳成功后可经直达按钮在正文页看到新内容

### Requirement: 毛玻璃克制视觉
前端 SHALL 采用毛玻璃（glassmorphism）视觉风格：深色基底 + 半透明磨砂层次，强调色克制、大面积留白。毛玻璃效果 SHALL 仅用于抽屉/卡片/证据栏等悬浮层，正文与聊天主体 SHALL 保持安静、不喧宾夺主。所有 UI 文案 SHALL 为中文。

#### Scenario: 悬浮层使用毛玻璃
- **WHEN** 渲染抽屉、卡片或证据栏
- **THEN** 这些悬浮层呈现毛玻璃磨砂质感
- **AND** 聊天主体与正文区域保持安静、不使用抢眼效果

#### Scenario: 全中文文案
- **WHEN** 任何 UI 元素展示文字
- **THEN** 文案为中文
